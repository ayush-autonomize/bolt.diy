# Changelog

## Added Azure OpenAI Provider
- **New File**: `app/lib/modules/llm/providers/azure.ts`
  - Implemented `AzureProvider` class extending `BaseProvider`.
  - Configured to use `AZURE_OPENAI_API_KEY`, `AZURE_OPENAI_ENDPOINT`, and `AZURE_OPENAI_DEPLOYMENT_NAME`.
  - Uses `@ai-sdk/openai` with custom fetch for API versioning (`2024-02-15-preview`).
- **Modified**: `app/lib/modules/llm/registry.ts`
  - Registered `AzureProvider` in the provider registry.
- **Modified**: `package.json`
  - Added `@ai-sdk/azure` dependency (though the implementation uses `@ai-sdk/openai`).

## Bug Fixes
- **Modified**: `app/utils/selectStarterTemplate.ts`
  - Fixed a bug where the `model` object was being passed directly to the API instead of the model name string.
  - This resolved a 404 error when using Claude 3.5 Sonnet.

## Scripts
- **New File**: `scripts/test-azure.ts`
  - Added a script to test Azure OpenAI integration.

## Anthropic Provider Updates
- **Modified**: `app/lib/modules/llm/providers/anthropic.ts`
  - Updated the list of static models to include newer versions available in the API:
    - Claude 4.5 Sonnet (`claude-sonnet-4-5-20250929`)
    - Claude 4.5 Opus (`claude-opus-4-5-20251101`)
    - Claude 3.7 Sonnet (`claude-3-7-sonnet-20250219`)
    - Claude 3.5 Haiku (`claude-3-5-haiku-20241022`)
  - Removed unavailable `claude-3-5-sonnet-20241022` model.
  - Corrected `maxCompletionTokens` for Claude 4.5 Sonnet to 64,000.
